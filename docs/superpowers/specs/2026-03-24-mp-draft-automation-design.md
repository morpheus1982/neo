# 微信公众平台草稿自动化设计文档

> 生成时间: 2026-03-24
> 状态: 待审核

---

## 概述

建立端到端自动化流程，使用 Neo 打开微信公众平台、检测登录状态、创建草稿、并生成/更新 API 文档。

## 目标

1. **自动化草稿创建** - 通过 UI 自动化完成草稿编写和保存
2. **捕获完整 API** - 在操作过程中捕获创建草稿的完整 API 请求（包含 POST body）
3. **生成 API 文档** - 更新现有的 `mp-api.md`，补充缺失的接口信息
4. **可复用流程** - 脚本可重复执行，方便后续扩展

## 背景

现有 `mp-api.md` 记录了 26 个微信公众平台接口，但创建草稿的核心接口缺少 POST body 结构信息。需要通过 UI 操作触发完整请求，由 Neo 捕获后补充文档。

## 技术方案

### 实现方式

**Shell 脚本** (`tools/mp-draft.sh`)

- 调用 neo CLI 命令完成各步骤
- 适用于 Git Bash / WSL 环境
- 简单直接，易于调试

## 执行流程

```
Step 1: 环境检查
  └─ neo doctor 验证 CDP 连接

Step 2: 打开页面
  └─ neo open "https://mp.weixin.qq.com"

Step 3: 检测登录状态
  ├─ neo snapshot 获取页面结构
  ├─ 未登录 → 提示扫码，循环等待
  └─ 已登录 → 继续

Step 4: 进入草稿编辑页
  ├─ 点击"草稿箱"菜单
  ├─ 点击"新的创作"按钮
  └─ 等待编辑器加载

Step 5: 填写草稿内容
  ├─ 填写标题
  ├─ 填写作者
  └─ 填写正文

Step 6: 保存草稿
  └─ 点击"保存为草稿"按钮

Step 7: 生成 API 文档
  ├─ neo schema generate mp.weixin.qq.com
  └─ 更新 mp-api.md

Step 8: 清理
  └─ 关闭标签页，输出摘要
```

## 命令行参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--title` | `测试文章 - YYYY-MM-DD` | 草稿标题 |
| `--content` | 固定测试文本 | 草稿正文 |
| `--author` | `测试作者` | 作者名 |
| `--skip-schema` | 否 | 跳过 schema 生成 |
| `--keep-tab` | 否 | 保持标签页打开 |

## 关键实现细节

### 登录状态检测

通过 `neo snapshot` 获取页面 a11y 树，检查是否存在以下元素判断登录状态：

- 未登录：存在"扫码登录"、"微信登录"等元素
- 已登录：存在用户头像、账号名称、"草稿箱"菜单等元素

### 等待扫码逻辑

```bash
# 循环检测，每 3 秒检查一次，最长等待 120 秒
for i in $(seq 1 40); do
  neo snapshot --json | grep -q "草稿箱" && break
  echo "等待扫码登录... ($i/40)"
  sleep 3
done
```

### 草稿编辑器交互

微信公众平台的编辑器是基于 iframe 的富文本编辑器，可能需要：

1. 切换到 iframe 上下文
2. 使用 `neo type` 或 `neo fill` 填写内容
3. 处理可能的动态加载延迟

### API 文档更新

脚本调用 `neo schema generate` 后，需要将 JSON schema 转换为 Markdown 格式，追加到 `mp-api.md`。

## 成功标准

1. 脚本能够成功创建一篇测试草稿
2. Neo 捕获到创建草稿的完整 API 请求（含 body）
3. `mp-api.md` 文档得到更新，新增或完善了草稿相关接口
4. 脚本可重复执行，不产生重复或冲突的草稿

## 风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|------|----------|
| 编辑器 UI 结构变化 | 中 | 使用灵活的 selector，基于文本内容匹配 |
| 登录态过期 | 低 | 检测到未登录时自动进入等待扫码流程 |
| 页面加载超时 | 低 | 使用 `neo wait` 确保元素就绪 |
| Neo 扩展断开 | 低 | 步骤 1 环境检查提前发现 |
